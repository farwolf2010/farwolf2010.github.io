# 调试

一直以来不好调试一直是weex的一大顽疾，手机上出了问题，真恨不得把头伸进去看看到底发生什么
不能打印也不能打断点，更看不了console的关键信息报错，只能toast，alert一点点调
这是官方的现状，那么如何方便的调试了？

  1. 程序务必兼容好web端，以便使用浏览器的f12调试,争取80%以上的业务错误通过浏览器端解决掉; 有同学要问我并不需要web版的为何还要兼容web，我想说的是,浏览器的f12稳定好用，强过weex-devtool百倍，weexplus统一了三端api就是为了更好的支持web，以便调试

  2. 对于web没问题手机端出问题的再使用weex-devtool

  3. 调用this.log('xx')方法，将信息输出到控制台

# 使用weex-devtool

使用要求：

  1. chorme版本必须小于等于79
  2. 不能开任何代理软件
  3. 虚拟机网卡也要禁用
